# Environment Variables Reference

## Overview

Open Notes uses a plain `.env` file to configure all services (FastAPI server, Discord bot, Playground). The `.env` file is read natively by Docker Compose and by `mise` (via `_.file = ".env"`).

Files:
- `.env.example` — Template committed to git. Copy this to get started.
- `.env` — Your local environment config. Gitignored; never commit real secrets.
- `.env.local` — Personal overrides (gitignored, highest priority).
- `.env.ci` — Sanitized CI values committed to git; no real secrets.

## Quick Start

1. `cp .env.example .env`
2. Generate JWT secret: `openssl rand -hex 32`
3. Generate Fernet key: `python -c 'from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())'`
4. Generate master encryption key: `python -c 'import secrets, base64; print(base64.urlsafe_b64encode(secrets.token_bytes(32)).decode())'`
5. Add Discord credentials from [Discord Developer Portal](https://discord.com/developers/applications)
6. Start services: `mise run dev:up`

## Migrating from .env.yaml

If you previously used `.env.yaml` files, rename them:

- `.env.yaml` → `.env`
- `.env.yaml.dev` → `.env.dev`
- `.env.yaml.staging` → `.env.staging`
- `.env.yaml.prod` → `.env.prod`

The format changed from YAML (`KEY: value`) to plain env (`KEY=value`). Quoted string values stay quoted. The `yq` tool and `_env:convert` mise task are no longer needed.

## Security Notes

- **Never commit** `.env` files containing real secrets.
- Generate `JWT_SECRET_KEY` with `openssl rand -hex 32`.
- Use strong, unique database passwords in production.
- Rotate secrets regularly.
- Use `.env.local` for personal developer overrides (gitignored, highest priority).
