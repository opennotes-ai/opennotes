# Public API Versioning Convention

OpenNotes exposes a dedicated URL prefix for the subset of the HTTP surface that third-party integrators (adapters) call. Everything outside that subset — admin, playground, simulation, internal webhooks — is implementation detail and may break without notice.

## The `/api/public/v1` prefix

- Canonical, stable, versioned prefix for adapter-contract routes.
- The legacy `/api/v1` and `/api/v2` prefixes are vestigial. Do not reference them in public-facing material (docs, examples, adapter SDKs).
- Every public route is **also** registered under its legacy prefix. In-the-wild Discourse installations still pointing at `/api/v2/...` keep working indefinitely; they just don't get the stable-contract guarantee.
- **The reverse is not true.** Legacy-only routes (admin, playground, simulation, webhook, registrar, scoring-analysis, etc.) stay legacy-only until a deliberate audit adds them to `PUBLIC_ADAPTER_ROUTERS`. Do not assume `/api/public/v1/<anything>` exists just because `/api/v2/<anything>` does.

## Source of truth: `src/public_api.py`

`PUBLIC_ADAPTER_ROUTERS` is the allowlist of routers that get double-registered under `API_PUBLIC_V1_PREFIX`. It is the authoritative answer to "is this endpoint public?" There is no scope-based auto-discovery — scopes are checked inside handler bodies, not as route metadata, so every addition must be deliberate.

### Adding a new public router

1. Write and land the router at its normal location (e.g. `src/<domain>/<name>_jsonapi_router.py`).
2. Register it under the legacy prefix in `src/main.py` exactly as you would any other route.
3. Audit: *does this endpoint make sense for any current or future adapter, not just one caller?* If the answer is "we only need it for Discord" or "only the playground calls this", do **not** add it to the allowlist. File a ticket describing the audit outcome before flipping it on.
4. Import the router in `src/public_api.py` and append it to `PUBLIC_ADAPTER_ROUTERS`.
5. Regenerate both OpenAPI artifacts: `mise run docs:generate:openapi` (produces `openapi.json` and `openapi-public.json`).
6. Add parity coverage in `tests/integration/test_public_prefix_parity.py` for at least one representative endpoint on the new router.

### Path-level allowlist for mixed routers

Some routers mix adapter-contract routes with self-service or admin routes (profiles_jsonapi_router owns `/user-profiles/lookup` alongside `/profiles/me`, `/profiles/{id}/opennotes-admin`, etc.). For those, `PublicRouterSpec` accepts a `path_allowlist` — only the listed paths get the public mount. `/profiles/me` and `/profiles/{id}/opennotes-admin` remain reachable at `/api/v2/...` (where they always were) but are deliberately NOT on the public surface.

### Why an allowlist instead of a scope filter?

Scopes on adapter-contract routers today are enforced at credential issuance, not per-request. Handlers call `is_service_account(current_user)` or no scope check at all, not `require_scope_or_admin(user, request, "platform:adapter")`. Introspecting FastAPI routes for scope metadata is unreliable, so an explicit list keeps "publicness" a code review decision rather than an emergent property. Hardening the public-prefix mount with an explicit `platform:adapter` dependency is tracked separately (see the pending audits section).

## Breaking changes

Paths, request bodies, and response shapes under `/api/public/v1` are part of the contract. Breaking changes require a new prefix (`/api/public/v2`) and a published deprecation window before `v1` is removed. Do not ship breaking changes under `/api/public/v1` — add new endpoints or new optional fields instead.

## OpenAPI artifacts

Two schemas live in the repo:

| File | Contents | Consumer |
| --- | --- | --- |
| `openapi.json` | Full schema (internal, playground, simulation, public). | Server developers; CLI/types generation for internal clients. |
| `openapi-public.json` | Filtered to `/api/public/v1/*` paths only. | Mintlify docs site (TASK-1460); third-party adapter SDK generation. |

Both are generated by `scripts/generate_openapi.py`. The public artifact is produced with `--public-only`. `mise run docs:generate:openapi` emits both in one step. `scripts/check_openapi_sync.py` validates both artifacts and is wired in three places:

1. Pre-commit hook `check-openapi-sync` (runs on changes to `opennotes-server/src/*.py`, `openapi.json`, or `openapi-public.json`)
2. CI job `validate-api-endpoints` in `.github/workflows/ci.yml` (runs on server or shared changes)
3. Local check: `mise run docs:check` from `opennotes-server/`

Any of these fails the build if either artifact is out of sync with the live FastAPI app.

## Pending audits

- `bulk_content_scan_jsonapi_router` — Discord bot currently uses this; semantics were informally intended to be universal but warrant review before committing to the public contract. Track in a follow-up task before adding to the allowlist.
- `scoring_jsonapi_router` — holds `/community-servers/{id}/scoring-analysis` which the Discourse admin dashboard calls on `/api/v2/...`. Not on the allowlist yet; an audit is required to decide which scoring endpoints are safe to advertise publicly.
- Profiles router: `/profiles/me`, `/profiles/{id}`, `/profiles/{id}/opennotes-admin` — mixed-router case above; `path_allowlist` currently keeps them internal. Re-evaluate once an explicit `platform:adapter` scope dependency lands on the public mount.
- Explicit `platform:adapter` scope dependency on the public-prefix `include_router` call itself — pending follow-up. Until then, the allowlist is the only guardrail on publicness.
