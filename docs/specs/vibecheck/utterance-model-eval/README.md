# Utterance Extractor Model Bake-off (TASK-1644)

Benchmarks `gemini-3-flash-preview` vs `gemini-3-flash-lite-preview` for
utterance extraction across short, medium, and long documents from production.
Model IDs are constants at the top of the script — edit them to compare other
pairs without changing the framework.

## Phases

| Phase | Subcommand | Free? | Description |
|-------|-----------|-------|-------------|
| 1 | `corpus` | Yes | Build bucketed manifest from exported rows |
| 2 | `run` | Vertex spend | Extract utterances with both models |
| 3 | `judge` | Vertex spend | Gemini Pro judges long+medium subset |
| 4 | `report` | Free | Aggregate results into markdown report |

## Input source

Export rows from `vibecheck_scrapes` via Supabase SQL editor or `pg_dump`:

```sql
SELECT
  id        AS scrape_id,
  normalized_url, url, host, page_kind, page_title,
  markdown, html, raw_html,
  screenshot_storage_key,
  scraped_at, expires_at, final_url, evicted_at
FROM vibecheck_scrapes
WHERE tier = 'scrape'
  AND expires_at > now()
  AND markdown IS NOT NULL
LIMIT 100;
```

Save as either:
- `.csv` (preferred — Supabase Studio's "Download CSV" from the SQL snippet view, no row-level truncation)
- `.json` (only safe for short rows — Studio's JSON row-view export silently truncates long text columns at 10243 chars)

The `corpus` and `run` subcommands accept either extension via `--input`.
Never commit the input file — it contains raw scraped markdown.

## Re-running

```bash
cd opennotes-vibecheck-server

# Phase 1: build corpus manifest (free, local only)
uv run python scripts/eval_utterance_extractor_models.py corpus \
    --input ~/Downloads/vibecheck_scrapes_rows.json

# Phase 2: run both models (burns Vertex)
uv run python scripts/eval_utterance_extractor_models.py run \
    --input ~/Downloads/vibecheck_scrapes_rows.json

# Re-run only failures:
uv run python scripts/eval_utterance_extractor_models.py run \
    --input ~/Downloads/vibecheck_scrapes_rows.json --force

# Phase 3: judge long+medium subset (burns Vertex)
uv run python scripts/eval_utterance_extractor_models.py judge

# Phase 4: generate report (free)
uv run python scripts/eval_utterance_extractor_models.py report
```

## Output layout

```
docs/specs/vibecheck/utterance-model-eval/
  corpus.json         # committed — IDs + lengths + buckets only
  README.md           # this file
  _judge_prompt.txt   # versioned judge prompt
  runs/               # gitignored — per-scrape model outputs
  judgments/          # gitignored — per-scrape judge verdicts
  REPORT.md           # gitignored — generated by report subcommand
```

## Screenshot tool

The `run` subcommand stubs `signed_screenshot_url` to always return `None`,
making the eval hermetic (no GCS calls). The agent can still call `get_html()`.
